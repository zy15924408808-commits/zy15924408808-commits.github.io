![[tiaoshi-1.jpg]]

>调试前要将Debug Information打开

![[tiaoshi-2.jpg]]
>将Optimization优化选项调为0，如果优化等级比较高的话，自己写的C语言程序和编译出来的代码可能不是等价的，调试过程中可能会出现执行顺序错乱的情况。

![[tiaoshi-3.jpg]]
>这个图标就是开始调试的按钮

![[tiaoshi-4.jpg]]
>第一个是reset按钮，点击之后就相当于将单片机复位。
>第二个是Run按钮，让单片机全速运行，有断点会直接运行到断点处停止。
>第三个是Stop按钮，让全速运行的单片机停止。
>第四个是Stap按钮，可以一步一步执行C语言代码。
>第五个是step over，可以直接将函数运行完成，不需要一步一步进函数里运行。
>第六个是step out，就是可以从函数里面跳出来。
>第七个是run to Cursor line，可以将代码直接运行到所指的位置。

![[tiaoshi-5.jpg]]
>第一个是Show NexT Statement，可以直接跳转到当前代码暂停的位置。

# 打断点
>按 Ctrl + B会出现你打的所有断点。如下图

![[tiaoshi-6.jpg]]
>有个Count的地方可以修改，第几次到这个断点的时候才停止。

# 显示变量和内存
![tiaoshi-7.jpg]]
>可以查看整个程序的调用栈和局部变量
>调用栈就是当前这个函数是怎么由之前的函数调用过来的。

![[tiaoshi-8.jpg]]
>查看全局变量，右键变量,添加到Watch中，就可以随时查看变量的状态。

![[tiaoshi-9.jpg]]
>这里可以查看某一块内存的情况。

# 访问断点和条件断点
![[tiaoshi-10.jpg]]
>在Watch中右击变量，然后点击 Set Access......

![[tiaoshi-11.jpg]]
>这里就可以设置一个读取断点和写入断点，就是代码运行到读取或写入这个变量的位置停止。

![[tiaoshi-12.jpg]]
>在这个表达式这里写等于100，就是在代码运行的过程中当变量等于100的时候，就停止。

>局部变量无法按照上述方法添加访问断点和条件断点
>方法如下：

![[tiaoshi-13.jpg]]
>在左下角使用&取地址符先得到变量的地址，复制下来
>然后点击Ctrl + B

![[tiaoshi-14.jpg]]
>然后将地址粘贴到表达式这里，就可以正常的访问断点了。

![[tiaoshi-15.jpg]]

>在表达式这里吧地址强制转化为指针然后再读取里面的值，然后再比较，就可以正常的条件断点了。
