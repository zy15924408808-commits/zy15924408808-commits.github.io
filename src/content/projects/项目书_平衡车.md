# 基于 STM32 的两轮自平衡车设计与实现

## 1. 项目简介

### 项目背景

两轮自平衡车是一种典型的倒立摆控制系统，对姿态感知、实时控制以及控制算法的稳定性要求较高。为了深入学习 STM32 嵌入式开发、传感器数据融合、PID 控制算法及电机控制技术，设计并实现了一套基于 STM32F103 的两轮自平衡车系统。

### 项目目标

实现以下功能：

- 小车自动保持直立平衡
    
- 根据编码器实现速度闭环控制
    
- 支持前进、后退及原地转向
    
- OLED 实时显示姿态角(俯仰角)、PWM、速度运行数据 
### 开发环境

- 主控芯片：STM32F103C8T6
    
- 开发工具：Keil MDK、VS CODE
    
- 配置工具：STM32CubeMX
    
- 下载工具：ST-Link
    
- 编程语言：C
    
- 调试工具：串口助手

## 4. 核心功能实现

### 4.1 姿态检测模块

功能描述：

通过 MPU6050 获取小车姿态角和角速度，为平衡控制提供反馈。

实现方式：

- I2C 通信读取传感器数据
    
- DMP 输出 Roll、Pitch、Yaw
    
- 获取 Gyro 数据用于微分控制
    

关键代码：

```c
mpu_dmp_get_data(&pitch,&roll,&yaw);
MPU_Get_Gyroscope(&gx,&gy,&gz);
```

---

### 4.2 直立环（PD 控制）

功能描述：

根据车体倾角计算电机输出，使车体恢复直立。

实现方式：

- Roll 角作为控制量
    
- GyroY 作为微分项
    
- 输出 PWM 控制电机
    

控制公式：

```text
PWM = Kp × (Angle - MidAngle)
    + Kd × Gyroscope
```

---

### 4.3 速度环（PI 控制）

功能描述：

利用编码器反馈控制车辆速度，实现静止及匀速运行。

实现方式：

- 编码器测速
    
- 低通滤波
    
- PI 控制
    
- 输出目标倾角
    

关键代码：

```c
    Velocity_out = Velocity(Target_Speed,encoder_left,encoder_right);
    Vertical_out = Vertical(Velocity_out + Med_Angle,roll,gyrox_lpf);
    Turn_out = Turn(gyroz,Target_turn);
    PWM_out = Vertical_out;
```

---

### 4.4 电机驱动模块

功能描述：

控制左右电机正反转及 PWM 输出。

实现方式：

- TIM PWM 输出
    
- GPIO 控制方向
    
- TB6612 驱动电机
    

关键代码：

```c
Load(MOTO1,MOTO2);
void Load(int moto1,int moto2)
{
    if(moto1 == 0)
    {
        HAL_GPIO_WritePin(GPIOB,GPIO_PIN_13,GPIO_PIN_RESET);
        HAL_GPIO_WritePin(GPIOB,GPIO_PIN_12,GPIO_PIN_RESET);
    }
    else if(moto1 < 0)
    {
        HAL_GPIO_WritePin(GPIOB,GPIO_PIN_13,GPIO_PIN_RESET);
        HAL_GPIO_WritePin(GPIOB,GPIO_PIN_12,GPIO_PIN_SET);
    }
    else
    {
        HAL_GPIO_WritePin(GPIOB,GPIO_PIN_13,GPIO_PIN_SET);
        HAL_GPIO_WritePin(GPIOB,GPIO_PIN_12,GPIO_PIN_RESET);
    }
    __HAL_TIM_SetCompare(&htim1,TIM_CHANNEL_4,abs(moto1));
    if(moto2 == 0)
    {
        HAL_GPIO_WritePin(GPIOB,GPIO_PIN_14,GPIO_PIN_RESET);
        HAL_GPIO_WritePin(GPIOB,GPIO_PIN_15,GPIO_PIN_RESET);
    }
    else if(moto2 < 0)
    {
        HAL_GPIO_WritePin(GPIOB,GPIO_PIN_14,GPIO_PIN_RESET);
        HAL_GPIO_WritePin(GPIOB,GPIO_PIN_15,GPIO_PIN_SET);
    }
    else if(moto2 > 0)
    {
        HAL_GPIO_WritePin(GPIOB,GPIO_PIN_14,GPIO_PIN_SET);

        HAL_GPIO_WritePin(GPIOB,GPIO_PIN_15,GPIO_PIN_RESET);
    }
    __HAL_TIM_SetCompare(&htim1,TIM_CHANNEL_1,abs(moto2));
}
```

## 5. 项目难点与解决方案

### 问题一：车辆无法保持平衡

问题现象：

小车能够站立，但会不断向前或向后移动，最终失去平衡。

分析过程：

1. 检查 MPU6050 安装方向
    
2. 校验 Roll、Pitch 轴对应关系
    
3. 调整机械中值（MidAngle）
    
4. 调整 PD 参数
    
5. 调整速度环 PI 参数
    

最终原因：

机械中值存在偏差，同时速度环积分参数过大导致持续输出。

解决方案：

重新标定机械中值，并重新整定串级 PID 参数，使车辆能够稳定保持平衡。

收获：

深入理解了串级 PID 控制及机械误差对控制系统稳定性的影响。

---

### 问题二：OLED 不显示问题
问题现象：

STM32F103 自制板上,OLED(I2C 接口)调用 `OLED_ShowNum` 显示 123456 无反应,屏黑。同一个屏在面包板上正常。

分析过程：
1. **先查代码**:main.c、i2c.c、oled.c、oledfont.h 全部通读——I2C1 重映射到 PB8/PB9、AFIO 时钟已开、初始化命令序列(含电荷泵 0x8D/0x14、开显示 0xAF)、ShowNum 逻辑、F8X16 字库都正确。**代码没问题,定位到硬件。**
    
2. **量电压(共地)**:VCC=3.3V ✓,SCL=3.3V ✓,**SDA=0.3V ✗**。SDA 空闲本该被上拉到 3.3V,却被拉低 → 锁定 SDA 线
    
3. **通断档**:SDA↔SCL 断开(无短路)✓;SDA↔GND 不响(无硬短路)。
    
4.  **面包板对比**:同屏在面包板好用 → 屏、代码、固件都没问题,故障是自制板 SDA 网络独有的。
    
5. **电阻档**:自制板 **SDA 对 GND = 1.1kΩ**,且网络上没接别的东西、**没有上拉电阻**。
最终原因：

OLED 模块自带约 10kΩ 上拉到 3.3V(所以 SCL 正常),而自制板 SDA 对 GND 存在一条约 **1.1kΩ 的意外漏电路径**。两者分压:

```
3.3V × 1.1k / (1.1k + 10k) ≈ 0.33V
```

正好等于实测的 0.3V。SDA 被死死压低,I2C 无法通信 → 屏不显示。这种 kΩ 级、非硬短路的漏电,典型来源是**焊接后的助焊剂/松香残留**在 SDA 焊盘与 GND 铺铜之间形成漏电。

收获：

**代码正确时优先查硬件**,用电压/电阻分层定位,能快速缩小范围。
**0.3V 这种"非 0 又非高"的电压,配合电阻档**能算出分压关系,直接指向漏电路径。
自制/手焊板,助焊剂残留和细铜丝搭桥是 I2C 线被拉低的高发原因。
I2C 是开漏总线,**必须有上拉**。本例靠模块自带上拉,建议自己板上给 SDA/SCL 各补 4.7k~10k 上拉,不依赖模块,更稳。
