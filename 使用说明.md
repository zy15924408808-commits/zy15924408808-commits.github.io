## 二、改完东西后，怎么让网站更新（最重要）

不管我改了什么（加文章、换证书、改文字），都是这「三步走」：

打开项目文件夹的命令行（或让 AI 帮我跑），执行：

```bash
git add -A
git commit -m "这次改了啥，随便写一句"
git push
```

推上去后，**等 1~2 分钟**，网址上就自动更新了。我什么都不用管。

> 网络连 GitHub 偶尔会超时失败，多试几次 `git push` 就行。

## 三、怎么发博客文章

我在 **Obsidian** 里正常写文章，写完把 `.md` 文件放进：

```
src/content/blog/
```

### 用文件夹分类
在 `blog/` 下面建子文件夹，文件夹名就是分类名：

```
src/content/blog/
├── 嵌入式/
│   └── 我的文章.md
├── 笔记/
│   └── 另一篇.md
└── 随手写.md          ← 不放文件夹的，归到「未分类」
```

### 文章开头要加几行（frontmatter）
就是 Obsidian 文档最顶上的属性：

```markdown
---
title: 文章标题
date: 2026-06-19
description: 一句话摘要（可选）
tags: [嵌入式, C语言]   # 可选
---

正文从这里开始，正常写就行。
```

### 注意
- **文件名不能重复**（不同文件夹里也不行），否则推送时会报错。
- 图片：放进 `public/blog-assets/`，文章里写 `![[图片名.png]]`。
- 内部链接：用文件名，比如 `[[rtos-scheduling]]`。

## 四、怎么加项目作品

在 `src/content/projects/` 新建一个 `.md` 文件：

```markdown
---
title: 项目名称
description: 一句话描述
order: 1                      # 排序，越小越靠前
stack: [STM32, C, FreeRTOS]   # 技术栈
repo: https://github.com/...  # 可选，代码链接
---

项目详细介绍。
```

## 五、怎么加获奖证书（显示在「关于」页底部）

两步：

1. 证书照片放进 `public/certs/` 文件夹。
2. 在 `src/content/awards/` 新建一个 `.md` 文件：

```markdown
---
title: 奖项名称
issuer: 颁发方           # 可选
date: 2025-10-15
level: 省级一等奖        # 可选
image: 我的证书.jpg      # 对应 public/certs/ 里的文件名
order: 1                # 可选，排序
---
```

访客点证书图片可以放大看。

## 七、想在电脑上先看效果（可选）

发布前想本地预览，在项目文件夹执行：

```bash
npm run dev
```

然后浏览器打开 http://localhost:4321 ，边改边看。看完按 `Ctrl+C` 关掉。

## 八、几个要记住的点

- **守好 GitHub 账号密码**，那是改网站的唯一钥匙。建议开两步验证（2FA）。
- 仓库是公开的：别人能「看」代码和文章源文件，但**改不了**。隐私内容别放进去。
- 不想发布的草稿：在 frontmatter 加一行 `draft: true`，它就不会出现在网站上。
- 搞不定的时候，把这份说明给 AI 看，让它帮我操作。
